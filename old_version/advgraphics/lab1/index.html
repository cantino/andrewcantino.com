Cocoa

For this lab, I decided to use Cocoa as my development language. Cocoa is an API for Macintosh written in Objective-C. Cocoa is powerful because so many features come ready-made, such as UI elements and extensive libraries. This was basically the first time that I had programmed in Cocoa, so it was a learning process. Here are some links about Cocoa:

  • Getting Started With Cocoa
  • Programming with Cocoa
  • Introduction to Developing Cocoa Objective-C Applications

Cocoa is very good at integrating with OpenGL. I found an excellent code example at NeHe, written by Bryan Blackburn. I used Bryan's OpenGL and Cocoa code as a starting point and modified it as needed. Here is a screen shot of my modifications to Bryan's original program. I added the utility window, the screen and frame capturing, multiple texture loading, and the ability to pause the display.

The Book and OpenGL
The purpose of this lab was to create an interesting scene in OpenGL. I started with this tutorial (and Bryan's code). After modifying the code to allow screen capturing and frame saving, I generated my first animation with OpenGL. The original tutorial code showed a textured box, so I decided to build from that point on, using the original box model, but reshaping it slightly and changing textures. I took 4 digital photos (with my brand new digital camera!) of a C/C++ book and used them to texture the object in OpenGL, generating a pretty realistic animation. The strange highlighting is a result of the flash from my camera embedded in the texture.

The Chair

Next, I decided to continue this project by photographing and modeling another object -- a chair. I wrote a function to generate a rectangular box in OpenGL, applying textures to each of it's sides, and then I called it multiple times to model the chair.
